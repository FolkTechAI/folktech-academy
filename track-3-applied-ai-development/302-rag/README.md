# Course 302: RAG (Retrieval-Augmented Generation)

**Track:** Applied AI Development
**Duration:** 4-5 hours
**Prerequisites:** Course 301
**Status:** Not Started

## What You'll Learn

By the end of this course you will:
- Understand why LLMs hallucinate and how RAG mitigates it
- Implement embedding-based retrieval from a document corpus
- Set up and query a vector database
- Choose the right chunking strategy for your content type
- Build a complete RAG pipeline from scratch
- Evaluate retrieval quality quantitatively
- Know when to use RAG vs fine-tuning

## Prerequisites

- Course 301: Prompt Engineering & System Design

## Course Content

Why models hallucinate, embedding-based retrieval, vector databases, chunking strategies, building a RAG pipeline from scratch, evaluating retrieval quality, RAG vs fine-tuning.

## Who This Is For

Developers who want to build AI features that answer questions accurately from their own documents and data — without hallucinating or going stale when the underlying data changes.
